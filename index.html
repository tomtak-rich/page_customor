Overview

시스템 개요

고객 알림톡 자동화 시스템은 철거의정석으로 접수된 고객이 견적 요청 → 상담 → 현장 방문 → 견적 확정 → 시공 → 완료 → 평가에 이르는 전 과정을 카카오 알림톡으로 자동 안내하는 내부 운영 도구입니다. 각 단계의 상태값 변화에 따라 정해진 템플릿이 자동 발송되며, ④ 방문 확인·⑧ 완료 확인은 고객 응답을 받아 후속 처리(재배정·일정 재설정 등)로 분기합니다.

①

고객 신뢰 가시화

진행 상황을 단계마다 먼저 알려, 고객이 "지금 내 공사가 어디까지 왔는지" 항상 확인할 수 있게 합니다. 불안 문의를 선제적으로 줄입니다.

②

운영 효율

반복 안내를 자동화해 본사·파트너의 수동 연락을 최소화합니다. 상태값 한 번 변경으로 다음 알림톡이 자동 발송됩니다.

③

본사책임제 증명

접수·계약·정산·완료 전 과정을 본사가 기록·통지하여, 책임 주체가 본사임을 고객에게 투명하게 남깁니다.

Architecture

시스템 아키텍처 (4층)

데이터·발송·자동화·포털 4개 층이 하나의 루프로 연결되어, 고객 응답이 자동으로 다음 알림톡을 발생시킵니다.

Layer 1 · Data

데이터

노션 / 에어테이블. 고객·현장·상태값·변수 원장 관리.

Layer 2 · Messaging

발송

루나소프트 알림톡 API. 템플릿 기반 카카오 알림톡 발송, 실패 시 SMS 대체.

Layer 3 · Automation

자동화

Make.com. 웹훅 수신, 상태값 분기, 다음 템플릿 트리거 오케스트레이션.

Layer 4 · Portal

포털

Softr. 고객·파트너 페이지(요청서·사진·견적서) 및 버튼 랜딩.

자동화 루프
버튼 클릭→ 포털 / 챗봇→ Make 웹훅→ 상태값 업데이트→ 다음 알림톡 자동 발송↺
Partner Track

파트너(업체) 트랙

검증 파트너 대표님을 위한 별도 알림 흐름입니다. 파트너의 사진 등록이 고객 알림의 트리거가 됩니다.

▣

신규 현장 배정

"대표님, 신규 현장이 배정되었습니다." 현장 주소·일정·요청 사항을 안내하고, 방문 일정 확인을 받습니다.

현장 정보 보기배정 확인
▤

사진 등록 요청

"대표님, 공사 진행 사진 등록을 요청드립니다." 포털에 진행 사진을 업로드하도록 안내합니다.

사진 등록하기

핵심 연동

파트너 사진 등록 = 고객 ⑦ '공사 진행 사진이 등록되었습니다' 알림 자동 트리거. 대표님이 포털에 사진을 올리면 Make 웹훅이 상태값을 갱신하고, 고객에게 ⑦ 알림톡이 즉시 발송됩니다.

추가금 0원 표기

'추가금 0원'은 광고 카피가 아닌 확정견적 정보성 표기로만 사용. 과장 표현을 피합니다.

④

변수 철자 일치

모든 #{변수명}은 데이터 원장과 철자가 정확히 일치해야 발송 오류가 없습니다. 대표번호 1555-6205는 부가정보에 고정.

STEP 01

템플릿 심사 등록

STEP 02

Make 트리거 매핑

상태값 ↔ 템플릿 발송 트리거를 Make.com에 매핑.

STEP 03

Softr 페이지 연결

요청서·견적서·사진 등록 등 버튼 랜딩 페이지 연결.

파일럿 운영
